## 背景
当前 UI 基于 Element Plus 搭配大量自定义样式，主要分布在各个 Vue SFC 的 `<style>` 与内联样式中。多数样式直接硬编码颜色（如 #409EFF、#3B82F6 等）、边框及字号，任何品牌或视觉调整都需要修改大量文件。

## 目标 / 非目标
- 目标：
  - 将所有设计令牌统一收敛至 `src/styles/variables.css`。
  - 建立可重复的迁移流程，把硬编码值替换为令牌。
  - 通过 lint/脚本等手段保障后续开发继续沿用令牌体系。
- 非目标：
  - 调整现有组件的布局或交互结构。
  - 引入新的 CSS 框架。

## 方案决策
- 继续使用挂载在 `:root` 上的 CSS 自定义属性作为令牌载体。
- 扩充令牌文件，覆盖语义状态色与当前项目中使用的排版刻度。
- 提供自动化脚本（如 `tools/apply_style_tokens.py`），用于批量替换并保持可复制流程。

## 风险与权衡
- 自动化可能遗漏运行时动态样式，仍需人工复核。
- 如果局部作用域样式覆盖了 `:root` 变量，可能引入视觉回归，因此关键页面需要 QA 冒烟确认。

## 迁移步骤
1. 确认并补全 `variables.css` 中缺失的令牌。
2. 对 `src/**/*.vue` 与 `src/styles/**/*.css` 执行自动化脚本。
3. 针对梯度、半透明等复杂场景手工收尾。
4. 补充文档，指导后续贡献者。
